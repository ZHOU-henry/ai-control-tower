# GitHub Flow

## Branching

- `main` stays releasable
- use short-lived branches per task
- merge through pull requests only

## Artifact Types

- Athena: plans, milestones, decision docs, backlog issues
- Hermes: research briefs, source maps, validation notes
- Hephaestus: code, scripts, tests, CI, implementation docs
- Themis: findings, risk registers, release gates, review decisions

## Merge Rule

No meaningful change should land without:

- a written scope
- a review path
- a provenance check when third-party material is involved
