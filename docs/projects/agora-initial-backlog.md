# Agora Initial Backlog

## Why Agora Goes First

`Agora` is the first product track because it can create the earliest real user-facing proof of value without depending on frontier robotics breakthroughs.

## Phase 1 Goal

Produce a credible open-source MVP boundary and the first implementation scaffold for an AI Agent platform.

## Phase 1 Outcomes

- a locked v0 boundary
- a first research brief grounded in real sources
- a provenance-aware release gate
- a concrete stack decision
- a first executable repo scaffold
- a first domain model for the core user journey
- a baseline CI and PR quality gate

## Workstreams

### Athena

- lock `Agora` v0 scope
- define success criteria
- translate strategy into backlog order

### Hermes

- validate pain points and prior art
- map adjacent open-source and market landscape
- identify source and license risks for borrowed patterns or code

### Hephaestus

- propose stack and repo structure
- scaffold the first runnable shape
- define core domain contracts

### Themis

- define provenance and release gate
- define what counts as acceptable external reuse
- define minimum PR and test expectations

## Execution Order

1. scope and success criteria
2. research brief
3. provenance and release gate
4. stack decision
5. scaffold
6. domain model
7. CI baseline

## Anti-Goals

- no chain or payment complexity in phase 1
- no TEE or enterprise-hardening as phase-1 blockers
- no false claims of general marketplace completeness
