# Contributing

## Operating Principle

This portfolio is open-source-first and provenance-aware.

That means:

- share aggressively
- copy carefully
- attribute correctly
- do not import unclear material just because it is convenient

## Workflow

1. Open or update an issue before major work.
2. Link research or prior art if the task depends on outside material.
3. Use a branch per task.
4. Open a pull request with scope, evidence, risks, and validation notes.
5. Pass the relevant quality gates before merge.

## Required Pull Request Checks

- clear scope
- test or validation notes
- license and provenance check for imported material
- risk notes for non-trivial changes
- reviewer signoff when required

## Third-Party Material Rule

- Do not copy code, datasets, images, models, or docs without a clear license.
- Preserve license headers, notices, and attribution when reuse is allowed.
- If the license is unclear, summarize and reimplement instead of copying.
